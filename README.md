### Philosophy

"Architecture over Implementation"
Code is the easiest part of engineering; the structure that supports it is what determines success. I focus on building systems that are resilient to change and easy to reason about.

"Writing code for humans, not just compilers"
Technical debt is often a result of poor communication. I prioritize self-documenting code, consistent naming conventions, and modularity to ensure that the code remains maintainable for the entire team.
